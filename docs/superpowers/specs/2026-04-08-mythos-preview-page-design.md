# Claude Mythos Preview ページ — 設計仕様

## 概要

Claude Mythos Previewの情報をまとめたストーリーテリング型のスクロールページ。「神話的・壮大」なビジュアルトーンで、パープル × シアンの配色。ネビュラ背景のヒーローから始まり、ベンチマーク比較、エピソード、安全性、価格情報へと読者を導く。

## 技術仕様

- **ルート:** `/mythos` → `src/app/mythos/page.tsx`
- **単一ファイル:** 1つの `page.tsx` に全コンポーネントを含める（既存ページと同じパターン）
- **ディレクティブ:** `"use client"`
- **スタイル:** CSS-in-JS（`<style>` タグ内 + インラインスタイル）
- **アニメーション:** IntersectionObserver ベースのスクロールリビール + CSS keyframes
- **外部依存:** なし（React + Next.js のみ）
- **フォント:** DM Sans + DM Mono（Google Fonts、既存ページと統一）

## 共通コンポーネント

### useReveal フック
IntersectionObserver で要素が画面に入ったかを検知。既存ページ（aice, best-practice）と同じパターン。

### R（リビール）コンポーネント
`useReveal` を使い、要素にフェードイン + translateY アニメーションを適用。`d`（ディレイ ms）で段階的に出現。

### Counter コンポーネント
数値を0からターゲットまでカウントアップするアニメーション。`requestAnimationFrame` + イージング。

## セクション詳細

### ① ヒーロー（フルスクリーン）

- **高さ:** `100vh`
- **背景:** ネビュラ演出
  - 3〜4個の紫のぼかし blob（`radial-gradient` + `filter: blur(30-50px)`）
  - CSS `@keyframes` でゆっくり漂う（8〜12秒サイクル）
  - 最下部にグラデーションフェード（次セクションへの接続）
- **コンテンツ（中央配置）:**
  - 「MYTHOS」— `font-size: 5rem`、`letter-spacing: 0.3em`、`font-weight: 800`、紫グロー付き `text-shadow`
  - 「Preview」— 小さめ、シアン、`letter-spacing` 広め
  - キャッチコピー：「Anthropic史上最も強力なAIモデル。一般公開はされない。」
  - スクロール誘導の小さなシェブロン（下向き矢印）
- **アニメーション:** タイトルはブラーリビール（blur → clear）、キャッチは 0.5s 遅延フェードアップ

### ② Mythosとは何か

- **レイアウト:** 中央寄せ、`max-width: 800px`
- **モデル階層図:**
  - 横並び4段階: Haiku → Sonnet → Opus → Mythos
  - 各モデルをボックスで表示、シアンのラインで接続
  - Mythos ボックスのみ紫グロー + やや大きく
- **テキスト:**
  - 「Opusの上位に位置するまったく新しいクラスのモデル」
  - 2026年4月7日、Anthropic公式ブログおよびレッドチームブログにて正式発表
  - Anthropic社内文書の引用:「これまでに開発した中で圧倒的に最も強力なAIモデル」

### ③ ベンチマーク比較

- **レイアウト:** 各指標が縦に並び、1つずつスクロールリビールで登場
- **指標ごとの構成:**
  - 指標名（シアン、uppercase、小さめ）
  - 説明文（1行、グレー）
  - 横バーチャート（2段：Opus 4.6 / Mythos Preview）
    - Opus: 暗めの紫 `#4c1d95` → `#7c3aed`
    - Mythos: 明るい紫 `#7c3aed` → `#a855f7` + `box-shadow` グロー
    - バーはスクロールトリガーで幅 0% → ターゲット% にアニメーション
    - 数値はバー内にカウンターで表示

- **5つの指標:**

| # | 指標名 | 説明 | Opus 4.6 | Mythos |
|---|--------|------|----------|--------|
| 1 | CyberGym | サイバーセキュリティ能力 | 66.6% | 83.1% |
| 2 | SWE-bench Verified | 実務コーディング | 80.8% | 「dramatically higher」（テキスト表示、バーなし） |
| 3 | Terminal-Bench 2.0 | 自律エージェント力 | 65.4% | 「dramatically higher」（テキスト表示、バーなし） |
| 4 | Exploit成功率 | Firefox 147 JSエンジン攻撃 | 2回 | 181回 |
| 5 | OSS-Fuzz Tier 5 | 完全な制御フロー乗っ取り | 0件 | 10件 |

- **「dramatically higher」の表現:** バーの代わりにグロー付きテキストブロック。`letter-spacing` 広め、パルスアニメーション。
