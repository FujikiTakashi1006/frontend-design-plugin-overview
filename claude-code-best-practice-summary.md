## 主な内容

### 1. コアコンセプトの解説

Claude Codeの機能を体系的に整理している。

| 機能 | 説明 |
|------|------|
| **Subagents** | 独立した実行環境で動く自律エージェント。カスタムツール・権限・モデル設定可能 |
| **Commands** | ユーザー起動型プロンプトテンプレート。既存コンテキストに知識を注入 |
| **Skills** | 設定可能で再利用可能な知識セット。事前読込・コンテキストフォーク対応 |
| **Hooks** | 特定イベントで実行される外部ハンドラー（スクリプト、HTTP、プロンプト等） |
| **MCP Servers** | 外部ツール・DB・APIへの接続プロトコル |
| **Memory** | CLAUDE.mdファイルと@pathインポートによる永続コンテキスト |
| **Settings** | 階層型設定システム。権限、モデル設定、サンドボックス等 |

### 2. プロンプティングのコツ

- **マイクロマネジメントしない** — バグ修正は「バグを貼って "fix" と言うだけ」でOK
- **不十分な修正が来たら** — 「今の知識で優雅な解決を再実装して」と言い直す
- **制約を伝える** — 「この変更をテストして合格するまでPRを作らないで」

### 3. 計画・設計のベストプラクティス

- **常にプランモードから開始する**（いきなりコード書かせない）
- 最小限のスペックを渡して **Claudeに質問させる**、その後新セッションで実行
- **段階的ゲート方式**: 各フェーズで単体テスト→自動化テスト→統合テストを挟む
- 複雑なタスクは Phase 分割して、各 Phase の完了条件を明確にする
